# XAM

**Приватный P2P голос/видео/чат для друзей. Без серверов, аккаунтов, телефонов и e-mail.**

Как Discord — но никакого центрального сервера: устройства соединяются напрямую, переписка и звонки идут между вами, а не через чужое облако.

## ✨ Что умеет

- 💬 **Личные чаты** — текст, картинки, видео, файлы, голосовые сообщения, реакции, markdown, поиск по переписке.
- 🟢 **«Печатает…», доставлено/прочитано** — как в нормальном мессенджере.
- 📞 **Звонки** — голос, видео, демонстрация экрана. Один-на-один и **групповые комнаты**.
- 🎮 **Оверлей звонка** поверх игр (как в Discord) — ники говорящих подсвечиваются, клики проходят сквозь.
- 🎙️ **Шумоподавление** (нейросеть RNNoise, локально), push-to-talk, подсветка говорящего.
- 👥 **Групповые комнаты** — общий текстовый чат + групповой звонок (mesh: каждый с каждым, без «хоста»).
- 📷 **Добавление по QR или ID** — навёл камеру и готово.
- 🔗 **Один аккаунт на телефоне и ПК** — перенос по QR, друзья синхронизируются автоматически.
- 🔔 **Уведомления в фоне** на Android — доходят, даже когда приложение свёрнуто (без push-серверов).
- 🎨 **Темы** (тёмная / стекло / светлая), акцентный цвет, свой фон.

## 🚀 Как начать

1. Запусти XAM, придумай ник и выбери тему — аккаунт создаётся **сам** (это криптографический ключ на твоём устройстве, без регистрации).
2. Открой **Профиль** → скопируй свой **ID** (или покажи **QR**).
3. Нажми **«Добавить»** → вставь ID друга (или отсканируй его QR) → отправь заявку.
4. Друг принимает — и вы на связи. Пишите, звоните, создавайте комнаты.

### Один аккаунт на двух устройствах
На ПК: **Настройки → Безопасность → «QR с ключом»**. На телефоне на экране входа: **«У меня уже есть аккаунт»** → отсканируй этот QR. Друзья и комнаты приедут на телефон сами.

## 🔒 Приватность и безопасность

- **Никаких серверов с вашими данными.** Переписка и звонки идут напрямую между устройствами (WebRTC, DTLS-шифрование).
- **Личность = пара ключей Ed25519.** Приватный ключ зашифрован средствами ОС и не покидает устройство.
- **Подмена друга невозможна** — при каждом соединении идёт криптографическая проверка владения ключом (challenge-response). Для полной уверенности сверь «код безопасности» в карточке друга.
- **Сообщения через друга-посредника** (когда прямое соединение не строится) запечатаны **end-to-end** (sealed box, X25519) и подписаны — посредник видит только шифртекст, прочитать или подделать не может.
- **Зашифрованный бэкап** аккаунта в один файл (AES-256-GCM + PBKDF2): без пароля файл бесполезен.
- Публичным узлам-посредникам видны только метаданные «кто-то ищет комнату» — **не содержимое**.

## 🛠️ Как это устроено (кратко)

XAM не использует центральный сервер. Друзья находят друг друга через публичные **Nostr-релеи** (по WebSocket — не блокируется там, где зарезан UDP), а дальше устанавливают **прямое WebRTC-соединение**. Весь трафик — текст, файлы, голос, видео — идёт P2P.

- **ПК** — Electron (Windows). **Телефон** — Capacitor (Android), общий код интерфейса.
- Стек: TypeScript, React, Zustand, Trystero (WebRTC + Nostr-сигналинг), Vite.
- Крипта: Ed25519 (подписи), X25519 sealed box (E2E поверх релея) — байт-совместимы между ПК и телефоном.

> Режутся публичные релеи у провайдера? **Настройки → Безопасность → «Связь»** — впиши свой релей (свой strfry на VPS / ПК с белым IP / в сети Tailscale).

## 🧑‍💻 Сборка из исходников

> Нужны Node.js 20+ и (для Android) Android Studio.

```bash
# ПК (Windows)
npm install
npm run dev                          # запуск в режиме разработки
npm run build                        # сборка кода
npx electron-builder --win --dir     # портативная сборка → release/win-unpacked

# Android (из мобильной копии проекта)
npm install
npm run build
npx cap sync android
cd android && ./gradlew assembleDebug   # APK → app/build/outputs/apk/debug
```

## ❓ Если что-то не работает

- **Друг «не в сети», хотя онлайн** — обновитесь оба до свежего релиза; первое соединение через публичные релеи может занять до ~30 секунд.
- **Заявки не доходят / не находите друг друга** — провайдер режет релеи. Укажите свой релей в Настройках → Безопасность.
- **Android убивает фон** — в настройках батареи телефона дайте XAM режим «Без ограничений», чтобы уведомления приходили при свёрнутом приложении.
- **Звонок виснет на «подключении»** — оба за «строгим» NAT; при необходимости укажите свой TURN в Настройках.
